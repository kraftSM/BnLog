## Планируемая структура БД для проекта (концептуально)
 -Users - Пользователи
  -Имя
  -Логин
  -пароль
 -Roles - роли
  - для группировки пользователей по правам доступа к элементам данных
    администраторы - руководят в группах
    модераторы - руководят в группах
    пользователи(зарестрированные)  пишут заметки и комментарии,загружать(?) ресурсы в свои(?) посты
    аноним(ы) могт просматривать открытые(?) посты и группы
 -Grups Группы прльзователей
    для группировки пользователем(-ми) своих постов вцелевые аудитории по интересам
 -Post заметки
  заметки пользователей
   только текст = по умолчанию
   в расширенном варианте может быть загружен и пред. отформатированный текст через ItemResurces
 -Tags Тэги(ярлыки)
  для построения группировки пользователем своих постов
 -Comments коментарии
  коментарии других пользователей под постами
 -ItemOptions доп.характеристики элементов
  типы и константы Global
  типы и константы User
  настройки пользователя, его статусы
  статистика постов (likes ...etc)  
 -ItemResurces доп.ресурсы для элементов 
  = то что (возможно) будет грузиться пользователем в binary формате
  посты в расширенном формате файлы
  загруженные файлы
